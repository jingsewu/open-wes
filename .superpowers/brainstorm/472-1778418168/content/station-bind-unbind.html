工作站绑定 & 解绑 UX 设计方案

两个核心场景：绑定（替换 SelectStation）+ 解绑（OFFLINE 后在 header 操作）

场景一：绑定工作站（替换 SelectStation）

进入 /wms/workStation 时，若没有绑定 → 显示绑定页面（全屏居中卡片，不是 overlay）

📌 绑定工作站 — 全屏居中
选择要绑定的工作站
绑定后方可开始作业，退出作业后可解绑
可用工作站
WS-001 拣选站 A
● 空闲中
WS-002 拣选站 B
● 空闲中
WS-003 拣选站 C
● 作业中（被占用）
不可用

场景二：工作站 Header — 作业中状态

正常作业时，header 右侧显示当前绑定的站点名 + "退出工作站"按钮（发送 OFFLINE 给服务端）

⚙️ 工作站 Header — ONLINE / NO_TASK 状态
作业工作站 WS-001 拣选作业 ● 作业中
↑ 作业中不显示"解绑"按钮，只能先退出

场景三：工作站 Header — 退出后（OFFLINE）

点击"退出工作站"→ 服务端 OFFLINE → header 变为下线状态，显示"解绑工作站"选项

🔴 工作站 Header — OFFLINE 状态
作业工作站 WS-001 ● 已下线
工作站已退出，可解绑后切换
↑ 点击"解绑工作站"→ 清除 localStorage stationId（clearStation()）→ 跳回菜单，显示绑定页面

完整用户旅程

首次进入：访问 /wms/workStation → 未绑定 → 显示绑定页 → 选站 → 确认 → 进入作业 UI
已绑定再次进入：访问 /wms/workStation → 有 stationId → 直接进入作业 UI（跳过绑定页）
退出并解绑：Header "退出工作站" → OFFLINE → Header 变红 → 点"解绑" → clearStation() → 跳回绑定页
退出不解绑：Header "退出工作站" → OFFLINE → 不点解绑 → 下次进入自动绑定同一台
